React的input中文输入下的onChange

Aug 11, 2017 | Web开发 | Hits

问题根源

打开百度首页，简单的一个搜索框，在其中输入一些东西，接口就会去请求server预拉取一些相关的关键词，列于输入框下供你选择。

以上的效果实现起来很容易，设想如果我们使用React实现，熟悉React的你是否已经想到了只需简单监听input的onChange事件，一旦监听到输入更改就执行一次api查询即可？理论上是的，事实上也确实可以做到，只是当你留意一下，你会发现如果我们输入开车上纽北这几个中文时，你多了很多次完全不正确的查询：比如发送了关键词：k，kaiche，kaichesha等等。

这完全不是我们想要的效果：我们期望的是，如果我们在输入中文，应该等待我们输完一个正确的中文后，再发送相关的请求。

你可能很疑惑，问题发生在哪里？原生的input不是这样的～

是的，如果你了解React，你已经猜到了这是React自身的合成事件的锅–它用的是自己的那套“非标准的”事件。

怎么解决

首先，使用非受控组件。所谓的非受控组件，对于input，简单来说，就是不要设置value。
